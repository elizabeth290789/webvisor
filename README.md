# Webvisor Session Triage

Streamlit-приложение для анализа сессий Яндекс Метрики и отбора записей Вебвизора, которые стоит открыть вручную в интерфейсе Метрики.

Приложение готово к запуску в **Streamlit Community Cloud**. Главный файл приложения — `app.py`.

## Демо-режим без токена

Если `YANDEX_METRIKA_TOKEN` не задан, приложение **не падает** и автоматически открывается в демо-режиме:

- показывает тестовые визиты и хиты;
- считает `score` по тем же правилам, что и для реальных данных;
- выводит заметное сообщение, что это демо-режим;
- позволяет посмотреть фильтры, таблицу, экспорт CSV и summary без локального запуска и без токена Яндекс Метрики.

Когда вы добавите `YANDEX_METRIKA_TOKEN`, приложение переключится на загрузку реальных данных из Yandex Metrica Logs API.

## Быстрый деплой в Streamlit Community Cloud

1. Откройте [Streamlit Community Cloud](https://share.streamlit.io/).
2. Нажмите **Create app** или **New app**.
3. В блоке выбора кода нажмите **Deploy a public app from GitHub** / **From existing repo**.
4. В поле **Repository** выберите GitHub-репозиторий с этим проектом: `webvisor`.
5. В поле **Branch** выберите ветку, которую хотите задеплоить, например `main`.
6. В поле **Main file path** укажите:

```text
app.py
```

7. Нажмите **Deploy**.
8. Если токен Яндекс Метрики еще не добавлен, приложение откроется с тестовыми данными в демо-режиме.

## Как подключить реальные данные Яндекс Метрики в Streamlit Cloud

Когда захотите вместо демо-данных подключить реальную Метрику:

1. Откройте ваше приложение в Streamlit Community Cloud.
2. Нажмите **Manage app**.
3. Откройте раздел **Settings**.
4. Перейдите в **Secrets**.
5. Добавьте секрет в формате TOML:

```toml
YANDEX_METRIKA_TOKEN = "ваш_oauth_token"
```

6. Нажмите **Save**.
7. Перезапустите приложение через **Reboot app** или дождитесь автоматического перезапуска.
8. После перезапуска в интерфейсе появится сообщение, что токен найден, и можно будет загрузить реальные данные по `counter_id`, `date_from` и `date_to`.

Опционально там же можно добавить ключ OpenAI для AI-summary:

```toml
OPENAI_API_KEY = "ваш_openai_api_key"
```

Если `OPENAI_API_KEY` не задан, кнопка «Сформировать выводы» построит обычный rule-based отчет без обращения к OpenAI API.

## Как получить токен Яндекс Метрики

1. Создайте OAuth-приложение в Яндекс OAuth.
2. Выдайте приложению права на чтение данных Метрики.
3. Получите OAuth-токен для пользователя, у которого есть доступ к нужному счетчику.
4. Добавьте токен в **Secrets** Streamlit Community Cloud как `YANDEX_METRIKA_TOKEN`.

## Что делает приложение

- Берет OAuth-токен из Streamlit Secrets или переменной окружения `YANDEX_METRIKA_TOKEN`.
- Если токена нет, автоматически включает демо-режим с тестовыми данными.
- Выгружает `visits` и `hits` через Yandex Metrica Logs API при наличии токена.
- Считает `score` от 0 до 100 для визитов, где пользователь был близок к конверсии, но не зарегистрировался.
- Показывает таблицу с фильтрами по устройству, URL, UTM, длительности, факту регистрации и score.
- Формирует AI-summary через OpenAI, если есть `OPENAI_API_KEY`; иначе использует rule-based summary.
- Экспортирует CSV: top проблемных визитов, все визиты со score, агрегаты по URL/device/UTM.


## Как понять, почему упал Streamlit Cloud

1. Откройте приложение в Streamlit Community Cloud и нажмите **Manage app**.
2. Перейдите в раздел **Logs**.
3. В логах ищите строки с маркерами `Failed`, `Traceback`, `ModuleNotFoundError`, `ImportError` или `SyntaxError`.
4. Скопируйте и пришлите последние 50 строк логов — обычно в них есть точная причина падения и файл, где оно произошло.

Перед деплоем также проверьте, что GitHub Actions workflow **Smoke test** прошел успешно: он компилирует основные Python-файлы и выполняет `python -c "import app"`, чтобы поймать ошибки импорта до Streamlit Cloud.

## Локальный запуск, если когда-нибудь понадобится

Локальный запуск не обязателен: проще использовать Streamlit Community Cloud. Но для локальной проверки можно выполнить:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

Без `YANDEX_METRIKA_TOKEN` локальная версия тоже откроется в демо-режиме.

## Как пользоваться с реальными данными

В интерфейсе укажите:

- `counter_id` счетчика Яндекс Метрики;
- `date_from` и `date_to`;
- ID целей регистрации через запятую, если нужно точно отделять зарегистрировавшихся пользователей.

Затем нажмите **Загрузить и посчитать**.

## Логика scoring

Правила добавляют баллы, если визит похож на потерянную конверсию:

- длительность больше 40 секунд без регистрации;
- 2+ просмотров страниц без регистрации;
- просмотр ключевых URL вроде `/promo/tasks-24/` или `/features/company/`;
- достижение важных URL вроде `/signup`, `/register`, `/demo`, `/checkout` без цели;
- мобильный пользователь ушел без цели;
- длинный визит без CTA click;
- события `form_start`, `form_error`, `modal_close`;
- `scroll_75` или `scroll_90` без CTA click;
- CTA click без регистрации.

## Ограничения

- Приложение **не скачивает записи Вебвизора автоматически**. Оно только отбирает `visitID` и сессии, которые стоит открыть вручную в интерфейсе Яндекс Метрики.
- Logs API готовит данные асинхронно: загрузка может занять минуты.
- Данные Logs API доступны с задержкой; текущий день может быть неполным или недоступным.
- Доступность отдельных полей зависит от счетчика, настроек Метрики и версии Logs API. Если поле недоступно, API вернет ошибку; в таком случае удалите или замените поле в `metrika_client.py`.
- Event/action-сигналы ищутся в доступных полях hits (`params`, URL, title, referer). Для надежного анализа убедитесь, что события CTA, форм и scroll передаются в Метрику.
