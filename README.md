# Webvisor Session Triage

Локальное Streamlit-приложение для анализа сессий Яндекс Метрики и отбора записей Вебвизора, которые стоит открыть вручную в интерфейсе Метрики.

## Что делает приложение

- Берет OAuth-токен из переменной окружения `YANDEX_METRIKA_TOKEN`.
- Выгружает `visits` и `hits` через Yandex Metrica Logs API.
- Считает `score` от 0 до 100 для визитов, где пользователь был близок к конверсии, но не зарегистрировался.
- Показывает таблицу с фильтрами по устройству, URL, UTM, длительности, факту регистрации и score.
- Формирует AI-summary через OpenAI, если есть `OPENAI_API_KEY`; иначе использует rule-based summary.
- Экспортирует CSV: top проблемных визитов, все визиты со score, агрегаты по URL/device/UTM.

## Как получить токен Яндекс Метрики

1. Создайте OAuth-приложение в Яндекс OAuth.
2. Выдайте приложению права на чтение данных Метрики.
3. Получите OAuth-токен для пользователя, у которого есть доступ к нужному счетчику.
4. Передайте токен приложению через переменную окружения:

```bash
export YANDEX_METRIKA_TOKEN="ваш_oauth_token"
```

## Переменные окружения

Обязательная:

```bash
export YANDEX_METRIKA_TOKEN="ваш_oauth_token"
```

Опциональная для AI-summary:

```bash
export OPENAI_API_KEY="ваш_openai_api_key"
```

Если `OPENAI_API_KEY` не задан, кнопка «Сформировать выводы» построит обычный rule-based отчет без обращения к OpenAI API.

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

В интерфейсе укажите:

- `counter_id` счетчика Яндекс Метрики;
- `date_from` и `date_to`;
- ID целей регистрации через запятую, если нужно точно отделять зарегистрировавшихся пользователей.

## Логика scoring

Правила добавляют баллы, если визит похож на потерянную конверсию:

- длительность больше 40 секунд без регистрации;
- 2+ просмотров страниц без регистрации;
- просмотр ключевых URL вроде `/promo/tasks-24/` или `/features/company/`;
- достижение важных URL вроде `/signup`, `/register`, `/demo`, `/checkout` без цели;
- мобильный пользователь ушел без цели;
- длинный визит без CTA click;
- события `form_start`, `form_error`, `modal_close`;
- `scroll_75` или `scroll_90` без CTA click;
- CTA click без регистрации.

## Ограничения

- Приложение **не скачивает записи Вебвизора автоматически**. Оно только отбирает `visitID` и сессии, которые стоит открыть вручную в интерфейсе Яндекс Метрики.
- Logs API готовит данные асинхронно: загрузка может занять минуты.
- Данные Logs API доступны с задержкой; текущий день может быть неполным или недоступным.
- Доступность отдельных полей зависит от счетчика, настроек Метрики и версии Logs API. Если поле недоступно, API вернет ошибку; в таком случае удалите или замените поле в `metrika_client.py`.
- Event/action-сигналы ищутся в доступных полях hits (`params`, URL, title, referer). Для надежного анализа убедитесь, что события CTA, форм и scroll передаются в Метрику.
